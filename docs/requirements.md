# 회의실 예약 시스템 요구사항 정의서

## 1. 프로젝트 개요

### 1.1 목적
기존 위키 기반의 수동 회의실 예약 관리를 대체하는 실사용 MVP 웹서비스를 구축하여 회의실 예약 프로세스를 자동화하고 효율성을 향상시킨다.

### 1.2 범위
- 회의실 정보 관리 (생성, 수정, 비활성화)
- 회의실 예약 관리 (생성, 조회, 취소)
- 사용자 권한 관리
- 예약 현황 타임라인 조회

## 2. 기능 요구사항

### 2.1 회의실 관리

#### 2.1.1 회의실 조회
- **FR-001**: 모든 사용자는 활성화된 회의실 목록을 조회할 수 있다.
- **FR-002**: 관리자는 모든 회의실(활성/비활성) 목록을 조회할 수 있다.
- **FR-003**: 회의실 정보에는 다음이 포함된다:
  - 회의실 이름
  - 건물
  - 층
  - 최대 수용 인원
  - 상태 (활성/비활성)

#### 2.1.2 회의실 생성
- **FR-004**: 관리자만 회의실을 생성할 수 있다.
- **FR-005**: 회의실 생성 시 다음 정보를 입력받는다:
  - 회의실 이름 (필수)
  - 건물 (필수)
  - 층 (필수)
  - 최대 수용 인원 (필수, 양수)
  - 상태 (기본값: 활성)

#### 2.1.3 회의실 수정
- **FR-006**: 관리자만 회의실 정보를 수정할 수 있다.
- **FR-007**: 수정 가능한 항목:
  - 회의실 이름
  - 건물
  - 층
  - 최대 수용 인원

#### 2.1.4 회의실 비활성화
- **FR-008**: 관리자만 회의실을 비활성화할 수 있다.
- **FR-009**: 비활성화된 회의실은 예약할 수 없다.
- **FR-010**: 미래 예약이 있는 회의실은 비활성화할 수 없다.
- **FR-011**: 회의실은 삭제하지 않고 비활성화 상태로만 변경한다.

### 2.2 예약 관리

#### 2.2.1 예약 조회
- **FR-012**: 모든 사용자는 예약 목록을 조회할 수 있다 (로그인 불필요).
- **FR-013**: 예약 조회 시 다음 필터링이 가능하다:
  - 회의실별 조회
  - 날짜별 조회
- **FR-014**: 예약 정보에는 다음이 포함된다:
  - 예약 제목
  - 회의실 정보
  - 예약자 정보 (이름)
  - 시작 시간
  - 종료 시간

#### 2.2.2 예약 생성
- **FR-015**: 로그인한 사용자만 예약을 생성할 수 있다.
- **FR-016**: 예약 생성 시 다음 정보를 입력받는다:
  - 회의실 선택 (필수)
  - 예약 제목 (필수)
  - 예약 날짜 (필수, 단일 날짜만)
  - 시작 시간 (필수)
  - 종료 시간 (필수)
- **FR-017**: 예약 시간은 30분 단위로만 선택 가능하다.
- **FR-018**: 예약 가능 시간은 08:00 ~ 19:00이다.
- **FR-019**: 예약은 같은 날짜 내에서만 가능하다 (시작일 = 종료일).
- **FR-020**: 토요일, 일요일에는 예약할 수 없다.
- **FR-021**: 공휴일에는 예약할 수 없다.
- **FR-022**: 점심시간(11:30 ~ 12:30)에는 예약할 수 없다.
- **FR-023**: 11:00 ~ 11:30 예약은 가능하다.
- **FR-024**: 동일 회의실의 예약 시간이 겹치면 예약할 수 없다.
- **FR-025**: 시작 시간은 종료 시간보다 이전이어야 한다.

#### 2.2.3 예약 상세 조회
- **FR-026**: 모든 사용자는 예약 상세 정보를 조회할 수 있다.
- **FR-027**: 예약 상세 정보에는 다음이 포함된다:
  - 예약 제목
  - 회의실 정보 (이름, 건물, 층, 최대 수용 인원)
  - 예약 시간 (시작 ~ 종료)
  - 예약자 정보 (이름)

#### 2.2.4 예약 취소
- **FR-028**: 로그인한 사용자는 본인의 예약을 취소할 수 있다.
- **FR-029**: 관리자는 모든 예약을 취소할 수 있다.

### 2.3 사용자 관리

#### 2.3.1 사용자 조회
- **FR-030**: 관리자만 사용자 목록을 조회할 수 있다.
- **FR-031**: 사용자 목록에는 다음 정보가 표시된다:
  - 이름
  - 사번
  - 부서
  - 이메일
  - 권한 (관리자/사용자)
- **FR-032**: 사용자 목록은 이름, 사번, 부서로 검색할 수 있다.
- **FR-033**: 사용자 목록은 권한별로 필터링할 수 있다 (전체/관리자/사용자).

#### 2.3.2 권한 변경
- **FR-034**: 관리자만 사용자 권한을 변경할 수 있다.
- **FR-035**: 권한은 "관리자"와 "사용자" 중 하나로 설정할 수 있다.

### 2.4 인증 및 권한

#### 2.4.1 로그인
- **FR-036**: SSO(Single Sign-On)를 통한 로그인이 가능하다.
- **FR-037**: 로그인 후 사용자 정보는 자동으로 시스템에 등록/업데이트된다.
- **FR-038**: 사용자 정보는 사번(employeeId)을 기준으로 관리된다.
- **FR-039**: 최초 로그인 시 사용자 정보가 자동으로 생성된다.

#### 2.4.2 로그아웃
- **FR-040**: 로그인한 사용자는 로그아웃할 수 있다.
- **FR-041**: 로그아웃 시 SSO 로그아웃 페이지로 리다이렉트된다.

#### 2.4.3 권한별 접근 제어
- **FR-042**: 관리자는 다음 기능에 접근할 수 있다:
  - 회의실 생성/수정/비활성화
  - 예약 생성/취소
  - 사용자 관리
- **FR-043**: 일반 사용자는 다음 기능에 접근할 수 있다:
  - 예약 생성/취소 (본인 예약만)
- **FR-044**: 비로그인 사용자는 다음 기능에 접근할 수 있다:
  - 회의실 목록 조회
  - 예약 목록 조회
  - 예약 상세 조회

### 2.5 화면 구성

#### 2.5.1 메인 화면
- **FR-045**: 메인 화면에는 다음이 표시된다:
  - 날짜 선택 달력
  - 회의실별 예약 타임라인
- **FR-046**: 타임라인에는 다음이 표시된다:
  - 시간대별 구분선 (08:00 ~ 19:00)
  - 예약 블록 (예약자 이름 표시)
  - 점심시간 표시 (11:30 ~ 12:30)
  - 공휴일 선택 시 "공휴일 예약 불가" 메시지
- **FR-047**: 달력에서 토요일, 일요일은 선택 불가능하다.
- **FR-048**: 달력에서 공휴일은 표시되지만 선택 불가능하다.

#### 2.5.2 회의실 관리 화면
- **FR-049**: 관리자만 회의실 관리 화면에 접근할 수 있다.
- **FR-050**: 회의실 관리 화면에는 다음 기능이 제공된다:
  - 회의실 목록 조회 (활성/비활성 모두)
  - 회의실 추가
  - 회의실 수정
  - 회의실 상태 토글 (활성/비활성)
  - 회의실 검색 (이름, 건물, 층)
  - 상태별 필터링 (전체/활성/비활성)

#### 2.5.3 사용자 관리 화면
- **FR-051**: 관리자만 사용자 관리 화면에 접근할 수 있다.
- **FR-052**: 사용자 관리 화면에는 다음 기능이 제공된다:
  - 사용자 목록 조회
  - 사용자 검색 (이름, 사번, 부서)
  - 권한별 필터링 (전체/관리자/사용자)
  - 권한 변경

## 3. 비기능 요구사항

### 3.1 성능
- **NFR-001**: 예약 목록 조회 응답 시간은 1초 이내여야 한다.
- **NFR-002**: 타임라인 화면 렌더링 시간은 2초 이내여야 한다.

### 3.2 보안
- **NFR-003**: 모든 API 요청은 적절한 인증 및 권한 검증을 거쳐야 한다.
- **NFR-004**: 사용자 정보는 사번을 기준으로 안전하게 관리되어야 한다.

### 3.3 사용성
- **NFR-005**: 직관적인 UI/UX를 제공해야 한다.
- **NFR-006**: 반응형 디자인을 지원한다 (데스크탑 우선).

### 3.4 호환성
- **NFR-007**: 최신 브라우저(Chrome, Edge, Firefox)를 지원한다.

## 5. 제약사항

### 5.1 기술적 제약
- 백엔드: Node.js + Express + TypeScript
- 프론트엔드: React + Vite + TypeScript + TailwindCSS
- 데이터베이스: PostgreSQL
- ORM: Prisma

### 5.2 비즈니스 제약
- 예약 시간은 30분 단위로만 가능
- 예약 가능 시간: 08:00 ~ 19:00
- 점심시간(11:30 ~ 12:30) 예약 불가
- 주말 및 공휴일 예약 불가
- 동일 회의실의 예약 시간 중복 불가
